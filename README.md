# projektpzpp
Wymagania funkcjonalne:
  - Rejestracja graczy - czyli mozliwość zarejestrowania/zalogowania się gracza w aplikacji
  - profil uzytkownika - bedzie mial za zadanie przechowywac informacje o uzytkowniku np jego stan konta
  - system zakladow - umozliwienie obstawiania i przyjmowania zakladow na wirtualna walute
  - system rankingowy - wyswietlanie rankingu graczy na podstawie ich wynikow
  - bezpieczne przetwarzanie danych zabezpieczenia przed oszustwami oraz bezpieczne przetwarzanie danych uzytkownikow

Wymagania niefunkcjonalne: 
  - Wydajność - czas odpowiedzi na kazde zadanie ponizej 2 sekund
  - skalowalnosc - mozliwosc obslugi wielu uzytkownikow jednoczesnie np do 100 graczy
  - bezpieczenstwo - zabezpieczenie danych uzytkownikow oraz implementacja mechanizmow przeciwdzialania oszustom
  - dostepnosc - aplikacja powinna byc dostepna 24/7
  - uzytecznosc - prosty i intuicyjny interfejs dla uzytkownika
